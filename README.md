### 🔭 Учебный проект №2: Автоматизация тестирования (GitHub)

**Название:** Автотесты для онлайн-кинотеатра «Кинопоиск» (UI + API)

**Задачи:** Разработать набор автотестов для проверки ключевого функционала онлайн-кинотеатра «Кинопоиск» (https://www.kinopoisk.ru/) и его публичного API (https://api.poiskkino.dev)

**Что сделано:**
- Реализованы UI-тесты (Selenium, POM) и API-тесты (Requests)
- Настроены маркеры pytest для раздельного запуска (`-m ui`, `-m api`)
- Создана модульная структура проекта (config/, pages/, api/, tests/)

**Выявленное ограничение:**
В процессе работы обнаружила, что UI-тесты блокируются капчей, которую автоматически обойти невозможно. Это показало, что для полной стабильности тестирования на данном проекте потребовалось бы:
- Договариваться с командой о тестовом окружении без капчи
- Либо фокусироваться на API-тестах как на основном слое проверки
